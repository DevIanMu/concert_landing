⚡ 双链路关键差异

登录态处理

  • 站外 H5：用户首次进入无登录态，需要手动登录。登录完成后重定向回 H5，需妥善处理 Token 传递和页面刷新。
  • App WebView：App 自动注入登录 Token（如通过 URL 参数或 JS Bridge），H5 无需额外登录步骤。

跳转 App 开户/入金

  • 站外 H5：未开户用户点击「前往开户」，需要先引导下载 App（或跳转应用商店），下载后完成开户，再手动返回 H5。链路长，流失率高。建议：跳转至中间页（"请在 App 内完成开户，完成后点击刷新"），并提供 Deep Link 直接唤起 App。
  • App WebView：直接通过 JS Bridge 唤起原生开户/入金流程，完成后自动返回 WebView，体验最流畅。

地址填写

  • 站外 H5：需要用户手动输入完整地址，建议增加香港地址自动补全（基于输入联想）。
  • App WebView：可通过 JS Bridge 读取用户在 App 内保存的地址簿，自动填充，减少输入成本。
