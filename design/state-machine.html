「盛夏煥新 · 港樂星夜」H5 状态机设计

状态机驱动单页式架构 | 一个 URL 适配全用户生命周期

0
访客态（未登录）
用户从站外渠道（短信/广告/社交媒体）或站内 Banner 进入 H5，尚未登录。此状态下可浏览完整活动规则，建立信任。
UI 渲染：活动 KV 头图 + 演唱会排期展示（可选场次列表公开预览，完整展示 VIP / 标准档，不可选）+ 权益说明（VIP 档 / 标准档）+ 活动规则面板（默认展开）+ 「登录 / 注册」按钮 + 已报名余量展示（公开数据）
核心操作：点击登录 → 唤起登录页（App 内直接跳转登录，站外跳转至登录中间页）

已登录未开户
用户已登录，但在云锋有鱼未完成开户（或开户流程未走完）。
UI 渲染：活动规则面板（默认展开）+ 当前状态提示卡片「您尚未完成开户，无法报名」+ 可选场次预览（完整展示 VIP / 标准档，完成开户并达标后可选）+ 「前往开户」按钮
核心操作：点击开户 → 跳转 App 开户流程（或站外唤起 App 下载）
Gate：完成开户
↓
2
已开户
用户已完成开户，账户总资产是否达标决定可报名的档位。资产卡片按分档规则提示：未达标标准档时提示距离标准档差额；已达到标准档但未达标 VIP 档时提示已达到标准档及距离 VIP 档差额；已达标 VIP 档时提示已达到 VIP 档。
UI 渲染：状态提示卡片（展示【账户总资产】完整金额并保留两位小数，与分档达标情况；未达标时卡片右下角显示「继续入金」按钮）+ 实时资产展示（每分钟刷新）+ 「立即入金」按钮（未达标时）或 「填写寄票地址」按钮（已达标时）+ 场次列表（完整展示 VIP / 标准档，已达标档位可选，未达标档位提示「未达标」）
核心操作：未达标时点击入金 → 跳转 App 入金页面 → 入金完成后返回 H5 自动刷新状态；已达标时选择场次与档位 → 填写寄票地址 → 确认提交
Gate：账户总资产 ≥ 10万 HKD（标准）或 ≥ 100万 HKD（VIP 档）
↓
4
已报名（履约中）
用户已成功报名并锁定名额，进入票务履约阶段。此状态为终态，活动周期内不可再报名。
UI 渲染：报名成功确认卡片 + 接下来会发生什么（门票制作 / 寄送门票 / 签收门票）+ 收件信息展示（收件人、地址、电话）+ 到场指引；不展示底部固定 CTA，不提供物流查询功能
核心操作：修改收件信息（截止日前）
终态：不可取消、不可改场次、不可二次报名
↓
5
已到场 / 已结束
演出结束后，用户状态归档。用于后续数据复盘和 ROI 计算。
UI 渲染：活动回顾页（感谢参与）+ 满意度问卷入口（可选）；不展示活动规则板块，不展示底部固定 CTA
归档态：数据沉淀用于复盘

✕
旁路：目标场次已售罄
用户在「已开户」状态下选择场次时，该场次已抢完。
UI 渲染：弹窗提示「该场次名额已售罄」+ 推荐其他可选场次（若有）+ 提示「其他场次仍有余量，可继续选择」
行为：返回场次列表，用户可重新选择其他场次。若所有场次均售罄，展示「名额已发放完毕」状态页。

关键设计原则

状态一次判定：用户进入 H5 时，后端并行查询「登录态 + 开户状态 + 资产状态 + 报名状态」，返回唯一状态码，前端据此渲染组件。
实时资产驱动：未达标用户的资产数据需要实时或准实时（建议 5 分钟内）同步，确保入金后返回 H5 能正确切换状态。
库存原子锁定：用户提交报名时，后端以原子操作（数据库乐观锁/Redis 分布式锁）锁定 1 张票，防止高并发超卖。锁定成功后生成报名记录。
3 周时间窗：距离演出不足 3 周的场次在 UI 上置灰并提示「报名已截止」，后端同时做校验拦截。
活动终身限报 1 场：后端在用户维度标记活动参与标识，已报名用户无论状态如何，进入 H5 直接渲染「已报名」状态。
